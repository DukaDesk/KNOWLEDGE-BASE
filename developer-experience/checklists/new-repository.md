# New Repository Checklist

Use this checklist when creating a new implementation repository.

## Naming

- [ ] Repository name follows conventions.
- [ ] Repository description added.

## Governance

- [ ] README.md created.
- [ ] AGENT_CONTEXT.md created.
- [ ] PROGRESS.md created.
- [ ] ARCHITECTURE_ALIGNMENT.md created.
- [ ] CONTRIBUTING.md created.
- [ ] CHANGELOG.md created.
- [ ] LICENSE added.
- [ ] .gitignore added.

## Structure

- [ ] src/ directory created.
- [ ] tests/ directory created.
- [ ] docs/ directory created.
- [ ] scripts/ directory created.
- [ ] .github/workflows/ directory created.

## Scripts

- [ ] scripts/bootstrap exists and is executable.
- [ ] scripts/build exists and is executable.
- [ ] scripts/test exists and is executable.
- [ ] scripts/lint exists and is executable.

## CI/CD

- [ ] Build workflow configured.
- [ ] Test workflow configured.
- [ ] Lint workflow configured.
- [ ] Security scan configured.
- [ ] Branch protection enabled.

## Boot Verification

- [ ] Clone succeeds.
- [ ] Bootstrap succeeds.
- [ ] Build succeeds.
- [ ] Tests pass.
- [ ] Lint passes.

## Documentation

- [ ] Build instructions in README.md.
- [ ] Test instructions in README.md.
- [ ] Run instructions in README.md.
- [ ] Contribution guide in CONTRIBUTING.md.
- [ ] Repository linked from developer-experience documentation.
