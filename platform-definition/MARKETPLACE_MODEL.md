## 1. Purpose

This document defines the DUKADESK Marketplace — the distribution platform for extensions that expand what DUKADESK can do.

---

## 2. Marketplace Categories

| Category | Description |
|----------|-------------|
| **Apps** | Complete applications or capability packages |
| **Themes** | Visual themes and design systems |
| **Components** | Reusable UI components for the runtime |
| **Templates** | Pre-built screen, workflow, or application templates |
| **Actions** | Reusable action handlers for the action system |
| **Workflows** | Pre-built automation workflows |
| **AI Agents** | Specialized AI agents for tasks and decisions |
| **Integrations** | Connectors to external systems |

---

## 3. Marketplace Participants

| Role | Description |
|------|-------------|
| **Creator** | Builds and publishes extensions |
| **Publisher** | Distributes extensions on behalf of creators |
| **Tenant** | Purchases and installs extensions |
| **Reviewer** | Evaluates extensions for certification |
| **Platform Operator** | Manages marketplace health and policy |

---

## 4. Marketplace Lifecycle

```
Create Extension
    ↓
Submit for Review
    ↓
Certification
    ↓
Publish
    ↓
Discovery
    ↓
Install / Purchase
    ↓
Updates and Support
    ↓
Deprecation or Removal
```

---

## 5. Certification Levels

| Level | Meaning |
|-------|---------|
| **Unverified** | Submitted but not reviewed |
| **Verified** | Passed automated checks |
| **Certified** | Passed manual review |
| **Trusted** | Long track record, premium support |
| **Platform Native** | Built and maintained by DUKADESK |

---

## 6. Marketplace Principles

1. Extensions are first-class platform citizens.
2. Discovery is based on quality, relevance, and trust.
3. Creators own support for their extensions.
4. The platform enforces security and compatibility standards.
5. Reviews and ratings inform discovery.
