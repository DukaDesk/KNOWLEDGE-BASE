# DUKADESK OS — Tests

This directory contains tests for repository-level validation and quality gates.

## Purpose

The DUKADESK OS repository is primarily a documentation and governance repository. Tests here validate:

- Required file presence
- Metadata completeness
- Cross-reference integrity
- Markdown formatting standards
- Repository bootstrap compliance

## Running Tests

Tests will be run via the CI workflow in `.github/workflows/`.

Local test commands will be added as scripts in `scripts/`.
