# Phase 06: Website

## Objectives

Build the public DUKADESK website and tenant app web experience.

## Prerequisites

- Phase 04 Backend Services complete or stable.
- SDK published.

## Deliverables

- Public marketing pages.
- Authentication pages.
- Tenant app web rendering.
- Documentation pages.

## Exit Criteria

- [ ] Website renders marketing content.
- [ ] Users can register and sign in.
- [ ] Tenant app is accessible via web.

## Dependencies

- Phase 04 Backend Services
- SDK

## Repositories Involved

- website/
- sdk/

## Specifications Required

- UI-0001 Public Website Foundation and Authentication Flows

## Verification Required

- Website E2E tests.
- SEO checks.
- Accessibility audit.

## Risks

| Risk | Mitigation |
|------|------------|
| SEO complexity | Use SSR or SSG |
| Performance | Optimize images and bundles |
