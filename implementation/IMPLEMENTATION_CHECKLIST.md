# Implementation Checklist

## Pre-Implementation Checklist

Before any repository starts implementation, confirm:

- [ ] Knowledge Base references are identified.
- [ ] Engineering Specifications exist and are approved or ready.
- [ ] Product Definition covers the relevant scope.
- [ ] UI Specifications exist (for UI repositories).
- [ ] Repository Bootstrap is complete.
- [ ] `AGENT_CONTEXT.md` is current.
- [ ] `ARCHITECTURE_ALIGNMENT.md` is current.
- [ ] Initial backlog of tasks exists.
- [ ] Milestone assignment is clear.

## Repository Startup Checklist

For each implementation repository:

- [ ] `README.md` is present and accurate.
- [ ] `AGENT_CONTEXT.md` is present and current.
- [ ] `PROGRESS.md` is initialized.
- [ ] `ARCHITECTURE_ALIGNMENT.md` is present.
- [ ] `CONTRIBUTING.md` is present.
- [ ] `CHANGELOG.md` is initialized.
- [ ] `scripts/bootstrap` runs successfully.
- [ ] `scripts/build` runs successfully.
- [ ] `scripts/test` runs successfully.
- [ ] `scripts/lint` runs successfully.
- [ ] CI/CD pipeline is configured.
- [ ] Branch protection is enabled.

## Per-Task Checklist

For each implementation task:

- [ ] Task references an engineering specification.
- [ ] Task is assigned to an implementation agent.
- [ ] Branch is created from latest target.
- [ ] Implementation follows repository conventions.
- [ ] Tests are added or updated.
- [ ] Self-verification is completed.
- [ ] Pull request links task and specification.
- [ ] Review is approved.
- [ ] CI checks pass.
- [ ] Documentation is updated.
- [ ] Task status is updated to Done.

## Phase Exit Checklist

Before exiting a phase:

- [ ] All phase deliverables are complete.
- [ ] All exit criteria are met.
- [ ] Integration tests pass.
- [ ] Architecture compliance is confirmed.
- [ ] Documentation is updated.
- [ ] Risks are reviewed.
