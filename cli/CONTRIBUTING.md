# Contributing to CLI

Thank you for contributing to the DUKADESK OS CLI.

## Getting Started

1. Read [AGENT_CONTEXT.md](AGENT_CONTEXT.md).
2. Read [ARCHITECTURE_ALIGNMENT.md](ARCHITECTURE_ALIGNMENT.md).
3. Pick a Task from [engineering-work](../engineering-work/).
4. Follow the [Developer Experience](../engineering-governance/developer-experience/) guides.

## Workflow

1. Create a branch from `main` or `develop`.
2. Implement the change with tests.
3. Run `scripts/test` and `scripts/lint`.
4. Update documentation and `PROGRESS.md` if needed.
5. Open a Pull Request linking the Task and Specification.

## Standards

- Follow CLI naming conventions.
- Provide tests for commands.
- Update help text and documentation.
- Reference Task and Specification IDs in commits.

## Review

All changes require review. See [CODE_REVIEW_GUIDE.md](../engineering-governance/developer-experience/CODE_REVIEW_GUIDE.md).

## Questions?

Ask in the team channel or escalate to the Engineering Lead.
