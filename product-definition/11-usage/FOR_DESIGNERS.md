# For Designers

## How to Use the Product Definition

As a designer, the Product Definition tells you **what to design and for whom** before any UI specification or Figma file is created.

## What to Design

For each persona and journey:

| Product Artifact | Design Output |
|------------------|---------------|
| Persona | Design constraints and mental model |
| User Journey | Flow diagrams, wireframes |
| Application | Screen maps |
| Feature | Screen designs, components |
| Role | Access-aware UI |

## Example Workflow

Product says:

> Sarah, a church administrator, needs to build and publish a member app.

Designer creates:

- User flow for app creation
- Builder Studio wireframes
- Mobile app screen designs
- Component library for church templates
