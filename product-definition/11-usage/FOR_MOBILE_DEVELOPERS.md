# For Mobile Developers

## How to Use the Product Definition

As a mobile developer, the Product Definition tells you **what screens, flows, and capabilities the mobile app must support**.

## Start Here

1. Read [PLATFORM_OVERVIEW.md](../01-platform/PLATFORM_OVERVIEW.md).
2. Read [PRODUCT_CATALOG.md](../02-products/PRODUCT_CATALOG.md) → Mobile App.
3. Read [APPLICATION_CATALOG.md](../03-applications/APPLICATION_CATALOG.md) → Mobile applications.
4. Read [USER_JOURNEYS.md](../06-users/USER_JOURNEYS.md).

## What to Build

For each application and feature assigned to you:

| Product Artifact | Mobile Output |
|------------------|---------------|
| Application | Screen group / navigation section |
| Feature | Screen, component, or user action |
| User Journey | End-to-end flow |
| Permission | UI visibility and access |
| Persona | UX decisions |

## Example Workflow

Product says:

> Customer opens app, browses catalog, adds to cart, checks out.

Mobile developer creates:

- `UI-00XX Mobile Catalog Screen`
- `UI-00XX Mobile Cart Screen`
- `UI-00XX Mobile Checkout Screen`
- Screen state management
- SDK client calls
- Tests
