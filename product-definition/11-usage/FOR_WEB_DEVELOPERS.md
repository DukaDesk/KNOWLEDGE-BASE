# For Web Developers

## How to Use the Product Definition

As a web developer, the Product Definition tells you **what web applications, pages, dashboards, and flows to build**.

## Start Here

1. Read [PLATFORM_OVERVIEW.md](../01-platform/PLATFORM_OVERVIEW.md).
2. Read [PRODUCT_CATALOG.md](../02-products/PRODUCT_CATALOG.md) → Website, Business Dashboard, Tenant Dashboard, Builder Studio.
3. Read [APPLICATION_CATALOG.md](../03-applications/APPLICATION_CATALOG.md).
4. Read [USER_JOURNEYS.md](../06-users/USER_JOURNEYS.md).

## What to Build

For each web product and application:

| Product Artifact | Web Output |
|------------------|------------|
| Product | Website or web app |
| Application | Page group / route section |
| Feature | Page, component, or interaction |
| User Journey | Multi-page flow |
| Role | Dashboard access and navigation |

## Example Workflow

Product says:

> Business owner signs in to Business Dashboard, manages products, and views orders.

Web developer creates:

- `UI-00XX Business Dashboard Shell`
- `UI-00XX Products Page`
- `UI-00XX Orders Page`
- Route configuration
- SDK integrations
- Tests
