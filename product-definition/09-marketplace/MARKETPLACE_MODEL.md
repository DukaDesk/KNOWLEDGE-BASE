# Marketplace Model

## Marketplace Purpose

The DUKADESK Marketplace allows tenants and developers to discover, purchase, and use extensions that enhance tenant apps.

## Marketplace Categories

| Category | Description | Examples |
|----------|-------------|----------|
| Templates | Pre-built app designs | Church template, Store template |
| Plugins | Functional extensions | Donation plugin, Booking plugin |
| Components | Reusable UI components | Hero carousel, Product grid |
| Integrations | Third-party service connectors | Payment gateway, Email service |
| Themes | Visual themes | Modern theme, Minimal theme |
| Workflows | Automation workflows | Welcome sequence, Abandoned cart |
| AI Agents | Specialized AI assistants | Support bot, Content helper |

## Marketplace Actors

| Actor | Role |
|-------|------|
| Publisher | Creates and publishes marketplace items |
| Tenant Owner | Browses and installs items |
| Platform Operator | Reviews and approves submissions |
| Customer | Benefits from enhanced tenant apps |

## Discovery Model

Tenants discover items by:

- Category
- Search
- Popularity
- Recommended for similar use cases
- Featured collections

## Cross-Tenant Marketplace

The marketplace can also act as a cross-tenant discovery surface:

- Tenants selling similar items can be grouped.
- Customers can browse stores by category or location.
- This transforms individual tenant apps into a broader ecosystem.

## Transaction Model

| Item Type | Transaction |
|-----------|-------------|
| Free | Direct install |
| Paid | Purchase through platform |
| Subscription | Recurring license |
| Commission | Platform takes a percentage |

## Governance

- Submissions are reviewed before publication.
- Items must meet quality and security standards.
- Publishers are accountable for support.
