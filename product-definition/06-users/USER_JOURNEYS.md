# User Journeys

## Core Journey: Business Owner Launches Tenant App

```text
Register
    ↓
Create Organization
    ↓
Create Tenant
    ↓
Choose Template or Start Blank
    ↓
Design App in Builder Studio
    ↓
Add Products / Content
    ↓
Configure Settings
    ↓
Publish App
    ↓
Share App with Audience
    ↓
Receive First Order or Interaction
```

## Core Journey: Customer Uses Tenant App

```text
Open App or Website
    ↓
Browse Catalog or Content
    ↓
Sign In or Sign Up
    ↓
Add Items to Cart
    ↓
Checkout
    ↓
Receive Confirmation
    ↓
View Order History
    ↓
Receive Notifications
```

## Core Journey: Administrator Manages Business

```text
Sign In to Business Dashboard
    ↓
View Dashboard Summary
    ↓
Manage Products
    ↓
View and Process Orders
    ↓
Manage Customers
    ↓
View Reports
    ↓
Send Notifications
    ↓
Adjust Settings
```

## Developer Journey

```text
Read Documentation
    ↓
Install SDK and CLI
    ↓
Build Extension or Integration
    ↓
Test Locally
    ↓
Submit to Marketplace
    ↓
Receive Approval
    ↓
Publish to Tenants
```

## Journey Priorities

| Priority | Journey | Reason |
|----------|---------|--------|
| 1 | Business Owner Launches App | Core value proposition |
| 2 | Customer Uses Tenant App | Validates commerce and engagement |
| 3 | Administrator Manages Business | Validates operations |
| 4 | Developer Builds Extension | Validates extensibility |

## Journey to Specification Mapping

Journeys are decomposed into features, which become engineering specifications.
