# Personas

## Primary Personas

### Sarah — Church Administrator

- **Role**: Organization Owner
- **Goal**: Create a mobile app for church members.
- **Needs**: Easy builder, event announcements, donations, member directory.
- **Tech skill**: Low. Needs no-code tools.
- **Journey**: Register → Create organization → Build church app → Invite members → Share app.

### Mike — Local Store Owner

- **Role**: Business Owner
- **Goal**: Sell products online and manage orders.
- **Needs**: Product catalog, cart, checkout, payments, order management.
- **Tech skill**: Medium. Comfortable with dashboards.
- **Journey**: Register → Create tenant → Add products → Publish store → Receive orders.

### Emily — School Office Manager

- **Role**: Content Manager
- **Goal**: Keep parents and students informed.
- **Needs**: Pages, announcements, calendar, media, notifications.
- **Tech skill**: Medium.
- **Journey**: Invite → Manage content → Send announcements → View engagement.

### Alex — Customer

- **Role**: End User
- **Goal**: Use a tenant app to buy, book, or engage.
- **Needs**: Simple sign-in, clear navigation, fast checkout, order history.
- **Tech skill**: High. Expects mobile app quality.
- **Journey**: Open app → Browse → Add to cart → Checkout → Track order.

### Jordan — Developer

- **Role**: External Developer
- **Goal**: Build plugins or custom integrations.
- **Needs**: SDK, CLI, documentation, marketplace.
- **Tech skill**: High.
- **Journey**: Read docs → Install CLI → Build plugin → Publish to marketplace.

## Secondary Personas

- **Support Agent**: Helps users with platform issues.
- **Platform Operator**: Monitors system health and tenant status.
- **Designer**: Creates templates and themes for the marketplace.

## Persona Usage

Personas guide feature prioritization, user journey design, and UI decisions.
