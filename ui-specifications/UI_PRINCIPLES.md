# UI Principles

## Core Principles

### 1. Clarity First

Every screen should make the next action obvious. Users should never wonder what to do next.

### 2. Consistency

Similar actions, components, and patterns behave the same way across all applications.

### 3. Progressive Disclosure

Show only what is needed at each step. Advanced options are available but not overwhelming.

### 4. Feedback

Every user action receives clear feedback: loading, success, error, or confirmation.

### 5. Efficiency

Common tasks require minimal steps. Power users can complete work quickly.

### 6. Forgiveness

Mistakes are easy to recover from. Destructive actions require confirmation.

### 7. Accessibility

Interfaces are usable by everyone, regardless of ability or device.

## Platform-Wide Behaviors

- All actions have hover, focus, active, and disabled states.
- Primary actions are visually prominent.
- Destructive actions use warning styling.
- Form submissions show validation inline.
- Loading states do not block navigation.
- Empty states explain what to do next.
- Error states are specific and actionable.

## Application Personality

DUKADESK is professional, approachable, and trustworthy. The UI should feel calm, organized, and responsive.

## Principle Checklist

When designing or reviewing a screen, ask:

- [ ] Is the purpose of this screen immediately clear?
- [ ] Is the primary action obvious?
- [ ] Are states handled explicitly?
- [ ] Is the language simple and direct?
- [ ] Does it work on all target devices?
- [ ] Is it accessible?
