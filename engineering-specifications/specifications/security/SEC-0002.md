# SEC-0002: Authorization and RBAC

## Identifier

SEC-0002

## Title

Authorization and Role-Based Access Control

## Purpose

Define how permissions are assigned to roles and enforced across DUKADESK platform and tenant apps.

## Problem

Different users need different levels of access. Platform admins, organization owners, tenant owners, managers, staff, and customers must only access resources and actions appropriate to their role.

## Scope

### In Scope

- Role definitions
- Permission catalog
- Role-to-permission mapping
- Permission enforcement in backend APIs
- Role-aware UI rendering

### Out of Scope

- Attribute-based access control (ABAC)
- Custom policy engine
- Fine-grained resource ownership beyond tenant scope

## Requirements

| ID | Requirement | Priority |
|----|-------------|----------|
| SR-01 | Platform roles are defined (Super Admin, Operator, Support) | Must |
| SR-02 | Tenant roles are defined (Owner, Manager, Staff, Customer) | Must |
| SR-03 | Permissions follow `<resource>:<action>` format | Must |
| SR-04 | Roles are assignable per tenant | Must |
| SR-05 | Backend APIs enforce permission checks | Must |
| SR-06 | Dashboards render UI based on role | Must |
| SR-07 | Missing permissions result in 403 Forbidden | Must |
