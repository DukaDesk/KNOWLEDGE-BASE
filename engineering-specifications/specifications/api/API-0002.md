# API-0002: Tenant Management API

## Identifier

API-0002

## Title

Tenant Management API

## Purpose

Define the API contract for creating, reading, updating, and managing tenants.

## Problem

Users must be able to create tenants, switch between tenants, and manage tenant settings. The API must enforce tenant isolation.

## Scope

### In Scope

- Create tenant
- List my tenants
- Get tenant by ID
- Update tenant settings
- Suspend tenant (admin)

### Out of Scope

- Multi-app per tenant
- Tenant deletion (soft delete only)
- Custom domain management

## Requirements

| ID | Requirement | Priority |
|----|-------------|----------|
| AR-01 | Authenticated user can create tenant | Must |
| AR-02 | Tenant creator becomes owner | Must |
| AR-03 | User can list tenants they belong to | Must |
| AR-04 | Tenant endpoints require tenant context | Must |
| AR-05 | Tenant owner can update settings | Must |
| AR-06 | Platform admin can suspend tenant | Should |

## API Contract

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | /api/v1/tenants | Create tenant |
| GET | /api/v1/tenants | List my tenants |
| GET | /api/v1/tenants/{id} | Get tenant |
| PUT | /api/v1/tenants/{id} | Update tenant |
| POST | /api/v1/tenants/{id}/suspend | Suspend tenant (admin) |

### Request: Create Tenant

```json
{
  "name": "My Church App",
  "slug": "my-church-app",
  "type": "church"
}
```

### Response: Create Tenant

```json
{
  "id": "ten_123",
  "name": "My Church App",
  "slug": "my-church-app",
  "type": "church",
  "ownerId": "usr_123",
  "createdAt": "2026-07-13T00:00:00Z"
}
```

### Errors

| Status | Code | Description |
|--------|------|-------------|
| 400 | INVALID_SLUG | Slug format invalid or taken |
| 403 | FORBIDDEN | User lacks permission |
| 404 | TENANT_NOT_FOUND | Tenant does not exist |
| 409 | SLUG_EXISTS | Slug already in use |
