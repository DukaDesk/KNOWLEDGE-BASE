# DB-0002: Tenant Schema

## Identifier

DB-0002

## Title

Tenant Schema

## Purpose

Define the database schema for tenants, tenant settings, and tenant-level configuration.

## Problem

DUKADESK is multi-tenant. Each tenant represents one app with its own data, branding, and settings. The schema must support this isolation.

## Scope

### In Scope

- Tenants table
- Tenant settings
- Tenant types/categories
- Tenant status

### Out of Scope

- Tenant-specific custom domains
- Regional placement metadata
- Billing subscriptions

## Data Model

### Entities

| Entity | Description | Relationships |
|--------|-------------|---------------|
| Tenant | Tenant app | Owner (User), Settings |
| TenantSettings | Flexible tenant configuration | Tenant |
| TenantType | Template category | Tenant |

### Schema

```sql
CREATE TABLE tenants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,
  slug VARCHAR(100) UNIQUE NOT NULL,
  type VARCHAR(50) DEFAULT 'generic',
  owner_id UUID NOT NULL REFERENCES users(id),
  status VARCHAR(50) DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE tenant_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  key VARCHAR(100) NOT NULL,
  value JSONB NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(tenant_id, key)
);

CREATE INDEX idx_tenants_slug ON tenants(slug);
CREATE INDEX idx_tenants_owner ON tenants(owner_id);
```
