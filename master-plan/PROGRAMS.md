# Programs

This document defines the programs that coordinate implementation work.

## Program Definitions

| Program | Scope | Lead Team | Wave |
|---------|-------|-----------|------|
| Foundation Program | Infrastructure, identity, backend foundation, CI/CD, developer platform | Team A, Team B, Team E | Wave 1 |
| Platform Engine Program | Runtime engine, builder engine, SDK, CLI | Team C, Team D | Wave 2 |
| User Surfaces Program | Mobile, website, dashboards | Team F, Team G, Team H | Wave 3 |
| Business Engine Program | Commerce, payments, orders, media, notifications | Team I | Wave 4 |
| Intelligence Program | AI platform, analytics, integrations, automation | Team J | Wave 5 |
| Hardening Program | Production readiness, performance, security, observability, scale | Platform Operations | Wave 6 |

## Program Responsibilities

Each program is responsible for:

- Defining scope and deliverables
- Managing cross-team dependencies
- Tracking progress against milestones
- Escalating blockers
- Delivering a working outcome at the end of the wave

## Program Interactions

Programs interact through:

- Shared specifications
- Dependency graph
- Master plan reviews
- Release trains

## Program Governance

- Programs are reviewed weekly during active waves.
- Major scope changes require master plan update.
- Program completion is gated by success criteria.
