# Pull Request Checklist

Use this template for every pull request.

## PR Information

- Title: `[<TASK-ID>] <description>`
- Linked Task: `<TASK-ID>`
- Linked Specification: `<SPEC-ID>`
- Repository: `<repository>`

## Author Checklist

- [ ] Implementation matches the specification.
- [ ] Tests added or updated.
- [ ] Self-review completed.
- [ ] Linting and formatting applied.
- [ ] Documentation updated.
- [ ] PROGRESS.md updated if applicable.
- [ ] No secrets committed.

## Specification Alignment

- [ ] Acceptance criteria addressed.
- [ ] Out-of-scope items not introduced.
- [ ] ADRs respected.

## Review

- [ ] AI review completed.
- [ ] Human review completed.
- [ ] Architecture check completed if required.
- [ ] All blocking comments resolved.

## Verification

- [ ] Unit tests pass.
- [ ] Integration tests pass.
- [ ] CI checks pass.
- [ ] Traceability links present.

## Merge

- [ ] Branch is up to date.
- [ ] Merge strategy selected.
- [ ] Source branch will be deleted.
