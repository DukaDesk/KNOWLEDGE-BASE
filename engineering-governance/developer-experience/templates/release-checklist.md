# Release Checklist

Use this template for every release.

## Release Information

- Version: `vX.Y.Z`
- Release Branch: `release/vX.Y`
- Release Manager: `@handle`

## Preparation

- [ ] All linked specifications are `Verified`.
- [ ] All linked tasks are `Done`.
- [ ] Version bumped.
- [ ] Changelog updated.
- [ ] Release notes drafted.

## Quality Gates

- [ ] All tests pass.
- [ ] Security scan passes.
- [ ] Dependency audit passes.
- [ ] Documentation updated.
- [ ] Architecture alignment confirmed.

## Deployment

- [ ] Deployed to Integration.
- [ ] Smoke tests pass in Integration.
- [ ] Deployed to Staging.
- [ ] Acceptance tests pass in Staging.
- [ ] Production deployment approved.
- [ ] Deployed to Production.

## Post Release

- [ ] Post-release verification completed.
- [ ] Monitoring alerts reviewed.
- [ ] Release announced.
- [ ] Rollback plan validated.

## Traceability

- Included specifications:
  - SPEC-0000
- Included tasks:
  - TASK-0000
