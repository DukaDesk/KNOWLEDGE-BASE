# New Feature Checklist

Use this checklist when starting a new feature.

## Specification

- [ ] Engineering Specification exists and is approved.
- [ ] Specification ID recorded in task.
- [ ] Acceptance criteria understood.

## Work Items

- [ ] Epic created and registered.
- [ ] Feature created and registered.
- [ ] Tasks created and registered.
- [ ] Task assigned to implementation agent.

## Branch

- [ ] Branch created from latest main/develop.
- [ ] Branch name follows convention.

## Implementation

- [ ] Code follows repository conventions.
- [ ] Tests added for new behavior.
- [ ] Existing tests still pass.
- [ ] Documentation updated.

## Verification

- [ ] Specification satisfied.
- [ ] Architecture aligned.
- [ ] Tests pass.
- [ ] Self-review completed.

## Pull Request

- [ ] PR title includes Task ID.
- [ ] PR description includes Specification ID.
- [ ] CI checks pass.
- [ ] Review requested.

## Merge

- [ ] All reviews approved.
- [ ] All comments resolved.
- [ ] Branch up to date.
- [ ] Source branch deleted after merge.

## Follow-Up

- [ ] Task status updated to Done.
- [ ] PROGRESS.md updated.
- [ ] Release notes updated if applicable.
