# Release Checklist

Use this checklist for every release.

## Release Planning

- [ ] Release version defined.
- [ ] Release branch created.
- [ ] Included specifications confirmed.
- [ ] Included tasks confirmed.

## Preparation

- [ ] Version bumped.
- [ ] Changelog updated.
- [ ] Release notes drafted.
- [ ] Rollback plan documented.

## Quality Gates

- [ ] Build passes.
- [ ] All tests pass.
- [ ] Lint passes.
- [ ] Security scan passes.
- [ ] Dependency audit passes.

## Environments

- [ ] Deployed to Integration.
- [ ] Smoke tests pass in Integration.
- [ ] Deployed to Staging.
- [ ] Acceptance tests pass in Staging.
- [ ] Performance baseline acceptable.

## Production

- [ ] Deployment approved.
- [ ] Deployed to Production.
- [ ] Post-release verification completed.
- [ ] Monitoring reviewed.

## Communication

- [ ] Release announced.
- [ ] Stakeholders notified.
- [ ] Incidents or issues logged.

## Closure

- [ ] Release branch merged or maintained.
- [ ] Release tagged.
- [ ] Documentation updated.
