# Success Metrics

This document defines the engineering KPIs tracked platform-wide.

## Engineering KPIs

```text
Architecture Compliance %
Specification Coverage %
Automated Verification %
Repository Health
Lead Time
Cycle Time
Change Failure Rate
MTTR
Technical Debt Trend
AI Contribution %
Documentation Coverage
```

## Metric Definitions

### Architecture Compliance %
Percentage of repositories passing architecture compliance review.

### Specification Coverage %
Percentage of implemented features covered by Engineering Specifications.

### Automated Verification %
Percentage of acceptance criteria verified by automated tests.

### Repository Health
Composite score of build, test, security, and documentation status across repositories.

### Lead Time
Time from specification approval to production deployment.

### Cycle Time
Time from task start to task completion.

### Change Failure Rate
Percentage of deployments causing incidents or requiring rollback.

### MTTR
Mean time to recovery from incidents.

### Technical Debt Trend
Change in open technical debt items over time.

### AI Contribution %
Percentage of tasks or code changes assisted or completed by AI agents.

### Documentation Coverage
Percentage of specifications, services, and repositories with current documentation.

## Reporting

Metrics are collected automatically where possible and reviewed in engineering governance meetings.

## Improvement

Underperforming metrics trigger improvement plans owned by the relevant Domain Lead or Repository Owner.
