# Website Architecture Alignment

This document records the architectural constraints and decisions that guide website development.

## Approved ADRs

| ADR | Title | Status |
|-----|-------|--------|
| | | |

## Design Principles

- Performance-first
- SEO-friendly
- Accessible
- Content-managed

## Patterns

- Static or server-side generation
- Component-based pages
- Image and asset optimization
- Analytics and consent-aware tracking

## Constraints

- Public pages must be optimized for Core Web Vitals.
- Forms must protect against spam and abuse.
- Content must be manageable without code changes.

## Alignment Verification

Before merging, confirm:

- [ ] Changes align with approved ADRs.
- [ ] Changes follow repository patterns.
- [ ] Changes respect listed constraints.
