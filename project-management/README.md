# Project Management

This directory contains project management artifacts for DUKADESK OS. It is intended for program managers, engineering leads, product managers, and stakeholders who need visibility into progress, plans, and decisions.

## Purpose

- Capture meeting notes, decisions, and action items.
- Track milestones, releases, and sprint plans.
- Maintain a PM-level backlog and roadmap.
- Provide alignment reports for leadership and external stakeholders.

## Directory Structure

```text
project-management/
  alignment-reports/   # Project status and steering reports
  backlog/             # PM-managed backlog and priorities
  meeting-notes/       # Meeting notes and minutes
  milestones/          # Milestone tracker and status
  release-plans/       # Release plans and checklists
  rfc/                 # Requests for comments / decision records
  roadmap/             # High-level roadmap
  sprint-plans/        # Sprint plans and retrospectives
```

## How to Use

1. **Meeting notes**: Create a new file for each meeting using `meeting-notes/TEMPLATE.md`.
2. **Status reports**: Update `alignment-reports/PROJECT_STATUS.md` weekly.
3. **Backlog**: Prioritize items in `backlog/BACKLOG.md` before sprint planning.
4. **Milestones**: Review `milestones/MILESTONE_TRACKER.md` in steering meetings.
5. **Roadmap**: Refer to `roadmap/ROADMAP.md` for long-term direction.
