# DUKADESK OS Roadmap

## Vision

Build a domain-agnostic, no-code/low-code platform that enables SMBs, churches, schools, and communities to create tenant apps, manage business operations, and distribute through a marketplace.

## Roadmap Summary

| Wave | Theme | Timeline | Focus |
|------|-------|----------|-------|
| Wave 0 | Engineering OS | Q3 2026 | Governance, specifications, planning stack |
| Wave 1 | Foundation | Q3 2026 | Infrastructure, identity, tenant, backend APIs |
| Wave 2 | Runtime & Builder | Q4 2026 | Runtime engine, visual builder, SDK, CLI |
| Wave 3 | User Surfaces | Q1 2027 | Mobile app, website, dashboards, shared UI |
| Wave 4 | Commerce & Services | Q2 2027 | Commerce, payments, orders, media, notifications |
| Wave 5 | Intelligence & Integrations | Q3 2027 | AI, analytics, integrations, automation |
| Wave 6 | Production & Scale | Q4 2027 | Hardening, performance, security, observability |

## Wave 0 — Engineering OS (Complete)

- DUKADESK Constitution
- Engineering governance consolidation
- Engineering specifications and work management
- Platform operations and execution governance
- Master plan and enterprise reference model
- Product definition and UI specifications
- Implementation planning repository

## Wave 1 — Foundation (Current)

### Objectives

- Bootstrap infrastructure repository
- Implement identity and tenant services
- Stabilize authentication and tenant APIs
- Build SDK for auth and tenant operations
- Establish CI/CD across all repositories

### Deliverables

- Deployable infrastructure to integration environment
- User registration, login, password reset
- Tenant creation, isolation, settings
- Auth API and tenant API contracts
- SDK with auth and tenant clients

### Success Criteria

- End-to-end authentication tests pass
- Tenant isolation tests pass
- CI/CD pipelines run on every repository

## Wave 2 — Runtime & Builder

### Objectives

- Build runtime engine for server-driven UI
- Create visual builder studio
- Publish and render app definitions
- Provide SDK and CLI for developers

## Wave 3 — User Surfaces

### Objectives

- Public marketing website with auth flows
- Mobile app with authentication
- Business dashboard shell and user management
- Tenant dashboard shell and settings

## Wave 4 — Commerce & Services

### Objectives

- Core commerce modules (products, categories, cart)
- Payment provider integrations
- Order lifecycle management
- Media and notification services

## Wave 5 — Intelligence & Integrations

### Objectives

- AI platform and prompt management
- Analytics and reporting
- External integrations and connectors
- Workflow automation

## Wave 6 — Production & Scale

### Objectives

- Production hardening and security audits
- Performance optimization
- Observability and incident response
- Marketplace launch
